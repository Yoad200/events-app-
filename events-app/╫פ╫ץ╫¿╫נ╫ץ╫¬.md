# 🎉 ניהול אירועים — הוראות התקנה

## מה זה?

אתר לניהול הרשמות של עובדים לאירועים. במקום WhatsApp:
- יוצר אירוע במחשב
- שולח קישור לקבוצה
- עובדים נרשמים
- אתה מאשר/דוחה בלחיצה

---

## 📋 מה צריך?

לפני שמתחילים — צריך לפתוח **3 חשבונות חינמיים**:

1. ✅ **חשבון GitHub** (יש לך כבר: Yoad200)
2. **חשבון Supabase** (חינם) — לדאטה בייס
3. **חשבון Vercel** (חינם) — לאחסון האתר

**זמן משוער: 30-45 דקות**

---

## 🚀 שלב 1 — Supabase (10 דקות)

### צור חשבון
1. לך ל: https://supabase.com
2. לחץ "Start your project"
3. הירשם עם **GitHub** (הכי קל)
4. אשר את האימייל אם צריך

### צור פרויקט
1. אחרי הכניסה — לחץ **"New Project"**
2. **Name:** `events-app`
3. **Database Password:** תיצור סיסמה חזקה ו**תשמור אותה!** (לא נשתמש בה אבל חשוב לשמור)
4. **Region:** Frankfurt (Europe Central) — הכי קרוב לישראל
5. **Plan:** Free
6. לחץ **"Create new project"**
7. ⏰ חכה 2-3 דקות שייווצר

### צור את הטבלאות
1. בסיידבר השמאלי לחץ **"SQL Editor"**
2. לחץ **"New query"**
3. **תפתח את הקובץ `schema.sql`** שיש בפרויקט שלי
4. **תעתיק את כל התוכן שלו** ותדביק בעורך
5. לחץ **"Run"** (או Ctrl+Enter)
6. אמור להופיע: ✅ Success

### העתק את הפרטים שצריך
1. בסיידבר לחץ **"Settings"** → **"API"**
2. **תעתיק** ושמור בצד את:
   - **Project URL** (לדוגמה: `https://xxxxx.supabase.co`)
   - **service_role** key (תחת "Project API keys", זה הארוך)

⚠️ **אל תשתף** את ה-service_role key עם אף אחד!

## 🚀 שלב 3 — Vercel (10 דקות)

### צור חשבון
1. לך ל: https://vercel.com
2. **"Sign Up"** עם **GitHub**
3. אשר את ההרשאות

### חבר את הפרויקט
1. בדשבורד של Vercel — **"Add New..."** → **"Project"**
2. תראה את ה-repos שלך מ-GitHub
3. ליד **`events-app`** — לחץ **"Import"**
4. **Framework Preset:** Other
5. **Root Directory:** השאר ריק
6. **Build Command:** השאר ריק

### הוסף את משתני הסביבה (Environment Variables) 🔑

לפני שלוחץ Deploy — צריך להגדיר את הסיסמאות. למטה בעמוד יש **"Environment Variables"**:

הוסף 5 משתנים:

| Name | Value |
|---|---|
| `SUPABASE_URL` | (Project URL שהעתקת מ-Supabase) |
| `SUPABASE_SERVICE_KEY` | (service_role key מ-Supabase) |
| `ADMIN_PASSWORD_HASH` | ראה למטה איך ליצור |
| `ADMIN_TOKEN` | אקראי, למשל: `abc123xyz789secret456` |
| `PASSWORD_SALT` | אקראי, למשל: `my_super_secret_salt_2026` |

### איך ליצור `ADMIN_PASSWORD_HASH`?

זה ה-hash של הסיסמה שלך לכניסה לאתר. תיצור אותו ב-3 צעדים:

1. לך לאתר: https://emn178.github.io/online-tools/sha256.html
2. בשדה "Input" — הקלד: `{הסיסמה_שלך}{PASSWORD_SALT}`

   לדוגמה, אם הסיסמה שלך `1234` וה-salt הוא `my_super_secret_salt_2026`:
   
   הקלד: `1234my_super_secret_salt_2026`

3. **תעתיק** את התוצאה הארוכה (64 תווים) ותשתמש בה בתור `ADMIN_PASSWORD_HASH`

⚠️ **חשוב:** ה-`PASSWORD_SALT` חייב להיות **בדיוק אותו ערך** ששמת בעמודה למעלה ובחיבור ההצפנה!

### Deploy!
1. עכשיו לחץ **"Deploy"**
2. ⏰ חכה 1-2 דקות
3. ייפתח האתר! 🎉

האתר שלך זמין בכתובת כמו:
```
https://events-app-xxxx.vercel.app
```

## 🆘 בעיות נפוצות

### "הגדרת השרת לא הושלמה"
חסר אחד ממשתני הסביבה. בדוק שכל 5 המשתנים מוגדרים ב-Vercel → Settings → Environment Variables.

### "סיסמה שגויה" אבל אני בטוח שזה נכון
בעיה ב-hash. תוודא שה-SALT שהשתמשת בו ליצירת ה-hash הוא **בדיוק** אותו אחד שב-`PASSWORD_SALT` ב-Vercel.

### "לא נמצא אירוע"
ה-share_id לא קיים בדאטה בייס. תוודא שבאמת יצרת את האירוע.

### האתר לא נטען
תבדוק ב-Vercel → Deployments → Logs. אם יש שגיאה, צלם ותשלח לי.
